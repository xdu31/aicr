01 · The problem

The reachability gap

AICR ships recipes across a large matrix of services, accelerators, operating systems, intents, and platforms. No single team can access every combination at all time — contributions for unreachable hardware either stall or get merged on trust alone.

DimensionValues
ServicesAKS · BCM · EKS · GKE · Kind · LKE · OKE
AcceleratorsA100 · B200 · GB200 · H100 · H200 · L40 · RTX PRO 6000
Operating systemsAmazon Linux · COS · RHEL · Talos · Ubuntu
Workload intentsInference · Training
PlatformsDynamo · Kubeflow · NIM · Run:ai · Slurm (Slinky)

Stalls

No maintainer has the cluster — or time to stand one up — to re-run the validation. The PR sits.

Merges on faith

aicr validate prints local output, but nothing a maintainer can cryptographically tie to who ran it.

02 · Produce

Validate → evidence

producer
# on VPN — validate + emit an unsigned bundle
$ aicr validate --recipe recipe.yaml --snapshot snapshot.yaml --emit-attestation ./out

# off VPN — sign (keyless OIDC), push, write the pointer
$ aicr evidence publish ./out --push ghcr.io/<owner>/aicr-evidence
03 · Sign

cosign → Rekor, the keyless way

What does that mean? Evidence is signer-identity-bound, not cluster-physicality-bound. It proves this identity signed this (recipe, snapshot, results, BOM) combo at this time — tamper-evident and publicly logged — not that the cluster physically existed. Closing that gap is a deferred follow-up. See Trust model section in ADR-007.
04 · Submit

Pointer files: locator, not payload

Push auto-derives a per-recipe tag (<slug>-<fingerprint>); verification pulls by digest, so the tag is only a label. git log on the pointer is the audit trail.

05 · Stay honest

The CI gate

Advisory now, required later Warning-only today — a hard gate with an empty evidence corpus would block every recipe PR. The mechanism is done; per ADR-007 it hardens to a required check once enough recipes carry evidence.

06 · The payoff

Tamper-evident

tampered bundle
$ sed -i 's/"failed"/"passed"/' summary-bundle/ctrf/deployment.json
$ aicr evidence verify ./summary-bundle

 2 signature-verify      passed   # the signed Statement is untouched…
 4 manifest-hash-check   failed   manifest inventory check failed for 1 file(s)

### Failed check details
- manifest-hash-check
  `ctrf/deployment.json` — sha256 mismatch (got 3f9a…, want 7b21…)

Verdict: bundle invalid — integrity check(s) failed (exit 2)

The signature still verifies, but the file no longer matches its manifest hash — and the manifest is bound to the signed predicate. Editing manifest.json to match instead breaks predicate.manifest.digest, which is inside the signature.
